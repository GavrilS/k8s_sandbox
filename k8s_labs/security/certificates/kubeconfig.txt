# Kubeconfig Lab
---

Q: Where is the default kubeconfig file located in the current environment?

A: Run the 'printenv' command and look for a variable called 'KUBECONFIG'. If it is missing 
then the default location is used for the kubeconfig file. This means the file is: 
'/root/.kube/config'. Else the kubeconfig file is the one listed by the 'KUBECONFIG' env 
variable.

---

are defined in it?

A: 4

A: 4

Q: What is the current context set to in the my-kube-config file?

A: test-user@development

---

Q:  would like to use the dev-user to access test-cluster-1. Set the current context to the 
right one so I can do that.

A: Run this command:

> kubectl config --kubeconfig=/root/my-kube-config use-context research

---

Q: We don't want to specify the kubeconfig file option on each kubectl command. Set the 
my-kube-config file as the default kubeconfig file and make it persistent across all 
sessions without overwriting the existing ~/.kube/config. Ensure any configuration changes 
persist across reboots and new shell sessions.

A: Update the ~/.bashrc file by adding this line at the end:

"export KUBECONFIG=/configs/my-kube-config"

Then run the following command:

> source ~/.bashrc
