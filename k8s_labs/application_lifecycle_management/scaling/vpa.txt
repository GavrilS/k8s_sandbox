# Steps to deploy the Vertical Pod Autoscaler:

1. Install VPA Custom Resource Definitions (CRDs)

VPAs custom resource definitions (CRDs) allow Kubernetes to recognize the custom resources 
that VPA uses to function properly. There is a sample VPA CRD definition file in 
'./vpa_resources/vpa_crds.yaml'. To deploy them we simply run the following command in k8s:

> kubectl apply -f ./vpa_resources/vpa_crds.yaml

---
2. Install VPA Role-Based Access Control (RBACs)

RBAC ensures that VPA has the appropriate permissions to operate within your Kubernetes 
cluster. There is a sample RBAC definition file in './vpa_resources/vpa_rbac.yaml'. 
To install the RBAC settings, run:

> kubectl apply -f ./vpa_resources/vpa_rbac.yaml

---
3. Clone the VPA Repository and Set Up the Vertical Pod Autoscaler

- clone the VPA repository:

> git clone https://github.com/kubernetes/autoscaler.git

- navigate to the VPA directory after cloning:

> cd autoscaler/vertical-pod-autoscaler

- deploy the VPA by executing the provided script:

> ./hack/vpa-up.sh

By following these steps, the Vertical Pod Autoscaler will be installed and ready to 
manage pod resources in your Kubernetes cluster.

#Lab questions:

Q: Which are the VPA CRDs that get installed as part of the Vertical Pod Autoscaler setup?

A: You can check by running the command:

> kubectl get -f ./vpa_resources/vpa_crds.yaml

---

Q: How many VPA deployments typically run in the kube-system namespace after installation?

A: There are 3 deployments. You can check with the following command:

> kubectl get deployments -n kube-system

and locate the ones starting with 'vpa-'.

---
Q: Check the 'vpa-updater-XXX' pod logs to see for any issues.

A: You can check the logs with the following command:

> kubectl logs vpa-updater-XXX -n kube-system

* change vpa-updater-XXX with the actual pod name

---

Q: Verify VPA recommendation for a deployment.

A: Run the following command:
> kubectl describe vpa <deployment-name>

# VPA cpu optimization lab

---
Monitoring Pod Resource Usage
To check the current resource usage (CPU and memory) of all running pods in your Kubernetes 
cluster, use the 'kubectl top pod' command. This command retrieves and displays resource 
metrics directly from the cluster's resource metrics API. It is particularly useful for 
tracking how efficiently your workloads are consuming cluster resources.

> kubectl top pod

---
There is a sample 'VerticalPodAutoscaler' configuration in './vpa_resources/vpa_cpu.yaml'. 
Once this is deployed to the kubernetes cluster we can check the cpu consumption with the 
following command:

> kubectl get vpa
